# BROWSER_MANUAL_TEST_CHECKLIST.md

## 測試目標
確認 `web/` 章節網站在桌面與手機寬度下可正常閱讀，且符合目前需求：

- 顯示 Chapter 1~10
- 章節頁結尾有 Google 圖書購書 CTA
- 版面清爽、可閱讀、可導航

## 測試前準備
1. 開啟 [web/index.html](/c:/ebooks/ai-python-automation/web/index.html)
2. 建議至少測 2 種視窗寬度：
   - 桌面：1440px 左右
   - 手機：390px 左右
3. 若使用瀏覽器開發者工具，請關閉快取後重整一次

## 1. 首頁檢查
- [ ] 頁面可正常開啟，沒有白屏或明顯錯版
- [ ] 頁首品牌名稱顯示為「AI 自動化工具書」
- [ ] Hero 區塊標題、說明文字、按鈕都可見
- [ ] 首頁文案有明確寫出目前已同步完整 10 章
- [ ] 首頁導覽中的 `全部章節`、`特色` 可正常跳轉
- [ ] `開始閱讀` 按鈕可進入 Chapter 1
- [ ] `瀏覽章節` 按鈕可跳到章節列表區塊
- [ ] `Google 圖書購買` 按鈕可開新分頁

## 2. 章節列表檢查
- [ ] 首頁章節卡片總數為 10
- [ ] 可看到 Chapter 1 到 Chapter 10
- [ ] 每張章節卡片都有標題與簡介
- [ ] 點擊每張卡片都能進入對應章節頁

## 3. 章節頁檢查
- [ ] `chapter.html?id=chapter01` 可正常顯示 Chapter 1
- [ ] `chapter.html?id=chapter02` 可正常顯示 Chapter 2
- [ ] `chapter.html?id=chapter03` 可正常顯示 Chapter 3
- [ ] `chapter.html?id=chapter04` 可正常顯示 Chapter 4
- [ ] `chapter.html?id=chapter05` 可正常顯示 Chapter 5
- [ ] `chapter.html?id=chapter10` 可正常顯示 Chapter 10
- [ ] 左側或上方章節導覽列出 10 個章節
- [ ] 目前章節在導覽中有明顯 active 狀態
- [ ] `回章節首頁` 按鈕可返回首頁
- [ ] 未帶 `id` 參數時，預設顯示 Chapter 1
- [ ] 若手動輸入不存在的 `id`，頁面仍能安全 fallback 到預設章節

## 4. CTA 檢查
- [ ] 任一章節頁結尾有「繼續閱讀」購書 CTA
- [ ] CTA 標題為「前往 Google 圖書購書頁」
- [ ] CTA 按鈕文字為「前往 Google 圖書」
- [ ] 點擊 CTA 會開新分頁
- [ ] CTA 連結導向你提供的 Google 圖書商品頁

## 5. 視覺與閱讀體驗檢查
- [ ] 整體色調為清爽淺藍風格
- [ ] 文字與背景對比足夠，沒有難以閱讀的區塊
- [ ] 卡片、按鈕、側欄的間距自然，沒有擠在一起
- [ ] Hero 區塊在桌面版不會過窄或過寬
- [ ] 章節閱讀頁段落間距舒適
- [ ] CTA 區塊有明顯視覺區隔，不會和正文混在一起

## 6. 響應式檢查
- [ ] 手機寬度下首頁沒有橫向捲動
- [ ] 手機寬度下章節卡片會正常換行
- [ ] 手機寬度下閱讀頁側欄會移到主內容上方或合理位置
- [ ] 手機寬度下按鈕仍容易點擊
- [ ] 手機寬度下文字不會過小

## 7. 基本互動檢查
- [ ] 所有首頁連結都可點擊
- [ ] 所有章節切換連結都可點擊
- [ ] 沒有明顯的 broken link

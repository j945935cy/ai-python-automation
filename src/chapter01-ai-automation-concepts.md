# 第 1 章：AI 自動化概念

## 這章要解決什麼問題
很多初學者一看到「AI 自動化」就會以為要先學一堆很難的語法，其實真正的起點不是語法，而是看懂工作流程。這一章要幫你建立一個核心觀念：

- 先看懂工作怎麼做
- 再拆成可以描述的步驟
- 最後才交給 Python 和 AI 幫忙

如果你每天都會做重複工作，例如下載資料、整理欄位、輸出報表，那就已經很適合開始學自動化了。

## 情境案例
假設你每週都要做一次銷售整理，流程可能像這樣：

1. 到系統下載 Excel
2. 打開檔案檢查欄位
3. 篩選大於指定金額的訂單
4. 複製結果到新的報表
5. 把摘要貼到訊息或 Email

這類工作有三個特徵：

- 會重複發生
- 步驟可以講清楚
- 做完之後可以檢查結果對不對

只要符合這三點，就很適合思考自動化。

## 概念說明
自動化的核心不是把所有事都交給機器，而是把「重複、可描述、可驗證」的工作交給程式處理。AI 的角色通常不是直接取代你，而是幫你拆流程、寫草稿、解釋錯誤與優化程式。

學習 Python 自動化時，先記住這個簡單模型：

1. 取得資料
2. 處理資料
3. 輸出結果
4. 驗證結果

很多工作場景都能放進這個模型，例如整理資料夾、篩選 Excel、抓網站資料、輸出週報表。

## 人工流程、Python 流程、AI 協作流程

### 人工流程
- 開啟檔案
- 找出需要的資料
- 手動整理
- 複製貼上到新檔案
- 再重新檢查

### Python 流程
- 讀入資料
- 依條件篩選或整理
- 自動輸出成新檔
- 顯示處理結果

### AI 協作流程
- 幫你把工作拆成步驟
- 根據需求寫出第一版程式
- 解釋每一段程式的用途
- 協助你修改錯誤或補上新需求

你可以把 AI 想成「協作助教」，不是完全代做的人。

## 程式範例
```python
tasks = ["下載資料", "整理欄位", "輸出報表", "檢查結果"]

for task in tasks:
    print(f"正在執行：{task}")

print("全部任務完成")
```

### 範例解讀
- `tasks` 是任務清單，也就是流程拆解後的結果。
- `for` 會依序執行每個步驟，模擬自動化流程一關一關往下走。
- `print()` 先用來顯示目前進度，這是很適合初學者理解流程的方式。

這段程式雖然還沒真的下載資料或輸出報表，但它先幫你建立一個很重要的習慣：把工作變成明確步驟。

## 輸入與輸出長什麼樣

### 輸入
- 一份工作需求
- 一串明確的步驟

### 輸出
- 可閱讀的流程
- 可逐步改寫成程式的任務清單

在真實專案裡，很多人一開始就急著寫程式，結果寫到一半才發現需求沒想清楚。先把輸入和輸出講清楚，後面會穩很多。

## 常見錯誤與排查

- 一開始就想全部自動化。
  建議先挑一小段重複流程，例如「讀取檔案並列出內容」。

- 流程講不清楚就開始問 AI。
  如果你自己都說不清楚，AI 很容易寫出看似完整、實際上不適用的程式。

- 沒有驗證結果。
  自動化不是跑完就好，還要確認結果是不是你要的。

- 把 AI 當成黑盒子。
  AI 可以幫很多忙，但你還是要知道程式的輸入、處理和輸出。

## AI 任務
請 AI 幫你把一個日常重複工作拆成 4 到 6 個步驟，並標示哪幾步適合用 Python 自動化、哪幾步需要人工確認。

## AI Prompt 範例
```text
你是一位 Python 自動化助教。
請把下面的工作流程拆成初學者可以理解的步驟，
並指出哪些部分適合用 Python 自動化、哪些地方建議人工確認。

工作內容：
每天下載資料、整理欄位、輸出報表、寄給主管。

請用條列方式回答，並保持白話。
```

## 練習題

### 基礎題
1. 列出 3 個你生活中可能重複發生的工作。
2. 判斷哪一個工作最適合自動化，並說明原因。

### 改寫題
1. 把其中 1 個工作拆成取得資料、處理資料、輸出結果、驗證結果四段。
2. 將你拆好的流程改寫成 Python 串列。

### 實作題
1. 參考本章範例，寫一段程式列出你自己的工作步驟。
2. 執行後確認每個步驟都有正確印出。

## 挑戰題
設計一個「每週資料整理」的自動化流程圖，先不用真的寫出完整程式，只要把步驟、輸入、輸出和檢查點列清楚即可。

## 小結
自動化的起點不是語法，而是流程。只要你能把步驟拆清楚，Python 和 AI 就能幫你把工作慢慢變成可執行的系統。這也是整本書最重要的核心能力。

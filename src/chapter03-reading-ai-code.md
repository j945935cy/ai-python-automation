# 第 3 章：看懂 AI 程式

## 這章要解決什麼問題
AI 可以很快產生程式，但真正重要的是你能不能看懂。只要你看不懂，就很難修改，也很難判斷結果到底對不對。

這一章的目標不是讓你一眼看懂所有 Python，而是建立一套穩定的閱讀順序。

## 情境案例
假設 AI 給你一段程式，說它會幫你篩選出大於 10 的數字。這時很多初學者會直接複製貼上執行，但更好的做法是先問自己：

1. 資料從哪裡來？
2. 程式在哪裡做判斷？
3. 結果存到哪裡？
4. 最後怎麼顯示？

只要先抓住這四件事，你就比較不容易被程式碼嚇到。

## 概念說明
閱讀 AI 程式最穩的方式，是先找出四件事：

1. 輸入資料是什麼
2. 處理邏輯在哪裡
3. 輸出結果去哪裡
4. 有沒有風險或看不懂的地方

不要一口氣讀完整個檔案，而是分段閱讀。你可以先把程式切成「資料準備」「主要處理」「輸出結果」三段，再逐段理解。

## 程式範例
```python
numbers = [5, 12, 18, 3]
result = []

for number in numbers:
    if number >= 10:
        result.append(number)

print(result)
```

### 範例解讀
- `numbers` 是輸入資料。
- `result` 是輸出容器，用來收集符合條件的結果。
- `for` 代表逐一檢查每個數字。
- `if number >= 10` 是條件判斷。
- `append()` 把符合條件的數字加進結果清單。
- `print(result)` 負責把最後結果顯示出來。

## 用固定順序閱讀程式

### 第一步：先看輸入
程式一開始通常會先建立資料、讀取檔案，或接收外部輸入。這段範例中的輸入是：

```python
numbers = [5, 12, 18, 3]
```

### 第二步：再看處理
找出迴圈、條件、函式呼叫等主要邏輯。這段範例的核心處理是：

```python
for number in numbers:
    if number >= 10:
        result.append(number)
```

### 第三步：最後看輸出
輸出不一定是 `print()`，也可能是寫入檔案、輸出 Excel、寄送訊息。這段範例的輸出是：

```python
print(result)
```

### 第四步：檢查風險
這段程式雖然簡單，但還是可以問：

- 如果資料是空的會怎樣？
- 如果條件要改成大於 15，要改哪裡？
- 如果想保留原本資料，又想另外輸出結果，現在的寫法合不合理？

這些問題會幫你從「看懂」進一步走到「能修改」。

## 人工流程、Python 流程、AI 協作流程

### 人工流程
- 自己先分段閱讀
- 用白話寫下每一段在做什麼
- 找出看不懂的地方

### Python 流程
- 確認輸入、處理、輸出
- 試著改一個小條件
- 觀察結果有沒有跟著改變

### AI 協作流程
- 請 AI 把程式切成小段
- 請 AI 解釋變數用途
- 請 AI 用更簡單的版本重寫
- 請 AI 比較修改前後差異

## 常見錯誤與排查

- 一次讀整段程式。
  先切小段再讀，比較不容易卡住。

- 只看語法，不看資料流向。
  初學者最重要的是知道資料怎麼進來、怎麼被處理、怎麼輸出去。

- 不敢修改。
  其實最好的學法就是改一個小地方，再看結果有沒有變。

- 完全相信 AI 沒有錯。
  AI 會寫程式，也會寫錯。能看懂才知道哪裡需要修。

## AI 任務
請 AI 將一段你看不懂的程式拆成 4 個小段，每段都用白話解釋，並指出初學者最容易卡住的地方。

## AI Prompt 範例
```text
請把下面這段 Python 程式拆成 4 個小區塊說明。
每個區塊都要回答：
1. 這段在做什麼
2. 為什麼需要它
3. 初學者最容易卡在哪裡
4. 如果我要自己修改，應該先改哪裡

請用白話說明，不要只講語法名稱。
```

## 練習題

### 基礎題
1. 找出範例中的輸入、條件與輸出。
2. 說明 `result = []` 的用途。

### 改寫題
1. 把條件改成 `number >= 15`，先預測結果，再實際執行確認。
2. 在迴圈中加入一行，印出目前正在檢查的數字。

### 實作題
1. 請 AI 幫你產生一段 10 行內的檔案處理程式。
2. 你自己把它切成三段，分別寫上註解：輸入、處理、輸出。

## 挑戰題
請讓 AI 產生一段含有迴圈與條件判斷的簡單程式，然後你自己用白話寫出這段程式的流程說明，不要直接照抄 AI 的解釋。

## 小結
真正的能力不是複製貼上，而是知道程式在做什麼。能看懂 AI 程式，才算真的會使用 AI。當你會看輸入、看處理、看輸出，你就開始具備修改與除錯的能力了。

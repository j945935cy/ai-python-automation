第 6 章：網頁爬蟲

這章要解決什麼問題

有些資料不是放在 Excel，而是放在網站上。像是公告、商品資訊、文章標題、價格列表，很多時候都需要先從網頁取得內容，才能進一步整理或分析。這就是網頁爬蟲的起點。

這一章的重點不是一開始就學很複雜的解析技巧，而是先建立一個穩定流程：

  • 發送請求
  • 確認有沒有成功拿到內容
  • 先讀懂 HTML
  • 再決定要抓什麼資料

情境案例

假設你每天都要查看某個網站的最新公告，手動流程可能是：

  1. 打開網站
  2. 等頁面載入
  3. 找到公告標題
  4. 複製到自己的筆記或報表

如果這件事常常重複，你就可以考慮先用 Python 把網頁內容抓下來。

概念說明

網頁爬蟲的基本步驟通常是：

  1. 發送請求
  2. 取得內容
  3. 擷取目標資料
  4. 存檔或分析

一開始先學會穩定取得 HTML，不急著一次用太多工具。只要你能先確認狀態碼、看懂回應內容，後續解析會容易很多。

人工流程、Python 流程、AI 協作流程

人工流程

  • 打開網站
  • 找到要看的資料
  • 複製內容
  • 貼到自己的檔案或報表

Python 流程

  • 發送 HTTP 請求
  • 確認狀態碼
  • 印出部分 HTML 內容
  • 再決定後續要抓哪些欄位

AI 協作流程

  • 幫你把想抓的資料拆成欄位
  • 解釋回應內容應該怎麼讀
  • 協助你從 HTML 找出標題、連結、價格等目標
  • 提醒你常見錯誤與網站使用限制

程式範例

import requests

url = "https://example.com"
response = requests.get(url, timeout=10)

print("狀態碼：", response.status_code)
print(response.text[:200])

範例解讀

  • requests.get() 會送出請求到指定網址。
  • timeout=10 可避免等待太久。
  • response.status_code 可判斷請求是否成功。
  • response.text[:200] 先顯示前 200 個字元，幫你快速確認拿到的是不是預期內容。

輸入與輸出長什麼樣

輸入

  • 一個網址
  • 想觀察的網頁內容
  • 一個明確目標，例如標題、價格或連結

輸出

  • 狀態碼
  • HTML 原始內容
  • 後續可繼續整理的文字資料

常見錯誤與排查

  • 網址寫錯。 先直接把網址貼到瀏覽器測試，確認網站能正常打開。
  • 狀態碼不是 200。 表示請求可能失敗，先不要急著解析內容，要先確認網站是否可連線。
  • 抓到的內容看起來很亂。 這很正常，因為 HTML 本來就不是給人直接閱讀的。先確認有拿到正確頁面，再慢慢挑出需要的部分。
  • 一開始就想抓太多欄位。 建議先抓一個最簡單的目標，例如頁面標題或前幾個連結。

AI 任務

請 AI 幫你把某個網頁的目標資訊拆成擷取步驟，例如標題、價格或連結，並說明應該先看哪些 HTML 特徵。

AI Prompt 範例

你是一位 Python 網頁爬蟲助教。
請教我如何用 requests 抓取一個網頁的 HTML，
並用最簡單的方式找出頁面中的標題文字。

限制：
- 適合初學者
- 先不要用太多工具
- 先解釋怎麼判斷請求有沒有成功

請用白話語氣說明，並列出常見錯誤。

練習題

基礎題

  1. 改成抓取你自己的測試網站。
  2. 印出回應內容長度。

改寫題

  1. 只有在狀態碼是 200 時才顯示內容。
  2. 加上一行，顯示目前抓取的是哪個網址。

實作題

  1. 抓取一個簡單頁面，印出狀態碼與前 200 個字元。
  2. 比較兩個不同網址的回應結果有什麼差異。

挑戰題

抓取一個列表頁面，輸出前 5 個標題。先用白話列出你打算怎麼抓，再請 AI 幫你補成第一版程式。

小結

爬蟲學習不要貪快。先把請求與回應看懂，再逐步學解析，會更穩定。當你能穩定取得網頁內容，後面不管要整理公告、收集資訊或做報表，都會順很多。

第 4 章：檔案自動化

這章要解決什麼問題

很多人第一次接觸自動化，最適合從檔案開始。原因很簡單：你看得到結果，也容易驗證對不對。像是建立資料夾、搬移檔案、依副檔名分類，都是很典型的入門場景。

這一章要幫你建立的能力是：

  • 看懂檔案整理的流程
  • 用 Python 取代重複搬檔、分類、建立資料夾的工作
  • 在真正搬移前先做好風險檢查

情境案例

假設你每天都會從瀏覽器下載很多檔案，下載資料夾很快就會變得很亂。你可能會手動做這些事：

  1. 找出所有圖片檔
  2. 建立一個 images 資料夾
  3. 把圖片搬進去
  4. 再確認有沒有漏掉

如果這件事常常發生，就很值得自動化。

概念說明

檔案自動化是最適合初學者的主題之一，因為你可以直接看到結果。常見需求有：

  • 建立資料夾
  • 批次搬移檔案
  • 依副檔名整理
  • 重新命名

pathlib 能讓路徑處理更直觀，也比較適合教學。你可以把它想成「用物件方式操作路徑」，比傳統字串拼接更好理解。

人工流程、Python 流程、AI 協作流程

人工流程

  • 打開資料夾
  • 找出指定類型的檔案
  • 建立目標資料夾
  • 一個一個搬移
  • 再檢查結果

Python 流程

  • 指定來源與目標資料夾
  • 找出符合條件的檔案
  • 建立目標資料夾
  • 逐一搬移
  • 印出處理結果

AI 協作流程

  • 幫你把資料夾名稱改成你的實際環境
  • 協助你加入安全檢查
  • 解釋 pathlib 每一行程式在做什麼
  • 幫你從「先列出檔案」進一步改成「真的搬移」

程式範例

from pathlib import Path

source_folder = Path("downloads")
target_folder = Path("images")
target_folder.mkdir(exist_ok=True)

for file_path in source_folder.glob("*.png"):
    file_path.rename(target_folder / file_path.name)
    print(f"已移動：{file_path.name}")

範例解讀

  • Path() 用來建立路徑物件。
  • mkdir(exist_ok=True) 表示若資料夾不存在就建立。
  • glob("*.png") 會找出來源資料夾中所有 PNG 檔。
  • rename() 在這裡可用來搬移檔案到新位置。
  • print() 讓你在執行過程中看到處理進度。

輸入與輸出長什麼樣

輸入

  • 一個來源資料夾
  • 一種副檔名條件，例如 *.png
  • 一個目標資料夾

輸出

  • 建立好的目標資料夾
  • 已搬移完成的檔案
  • 畫面上的處理提示

常見錯誤與排查

  • 來源資料夾不存在。 先確認 downloads 真的存在，或改成你自己的資料夾名稱。
  • 目標資料夾名稱寫錯。 如果路徑打錯，檔案可能搬到不是你預期的位置。
  • 一開始就直接搬移，有風險。 初學者可以先把 rename() 改成 print(file_path.name)，先確認找到的是不是正確檔案。
  • 檔名衝突。 如果目標資料夾裡已經有同名檔案，後續最好再加入檢查。

AI 任務

請 AI 按照你的實際資料夾名稱，改寫成可直接使用的版本，並提醒可能的風險與安全檢查方式。

AI Prompt 範例

你是一位 Python 檔案自動化助教。
請用 pathlib 幫我寫一個 Python 程式，
把 downloads 資料夾中的所有 PDF 搬到 reports 資料夾。

限制：
- 適合初學者
- 先提供簡單版本
- 再提醒我有哪些風險

輸出格式：
- 先給完整程式
- 再逐段解釋
- 最後列出常見錯誤

練習題

基礎題

  1. 把範例改成搬移 .jpg 檔。
  2. 建立 data、output、backup 三個資料夾。

改寫題

  1. 只列出資料夾中的所有檔案，不做搬移。
  2. 把目標資料夾名稱改成 pictures。

實作題

  1. 設計一個小程式，把某個資料夾中的 .txt 檔搬到 texts 資料夾。
  2. 執行後確認檔案真的被搬移，並且畫面有正確提示。

挑戰題

設計一個依副檔名分類的檔案整理器，把檔案分到 images、docs、archives 三類，並先用 print() 模擬分類結果，再考慮真正搬移。

小結

檔案自動化雖然簡單，卻非常實用。當你學會這一章，就已經能開始解決真實工作的痛點。更重要的是，你會開始習慣先想流程、再寫程式。

第 10 章：完整專案

這章要解決什麼問題

學到這裡，你已經接觸過流程拆解、Prompt、讀懂 AI 程式、Excel 處理、爬蟲、自動化操作與報表輸出。最後一章的重點不是再學一個新工具，而是把前面的能力串成一個真的能交付的小專案。

這一章要完成的專案是：

  • 讀取每週訂單資料
  • 統計各地區銷售金額
  • 產出摘要報表
  • 產生主管看得懂的文字結論

你會看到，真實工作通常不是單一技巧，而是多個小流程接在一起。

專案情境

假設你每週五都要整理一份銷售資料，交給主管做會議使用。原本的工作流程可能是：

  1. 打開 weekly_orders.xlsx
  2. 檢查欄位是否正確
  3. 依地區統計總金額
  4. 找出本週銷售最高地區
  5. 另存摘要 Excel
  6. 寫一段簡短中文說明

如果每週都重做一次，這就是非常典型的自動化情境。

概念說明

完整專案的重點，不是把所有功能寫得很複雜，而是把前面學過的能力合理串起來。你可以把它看成一個小型工作流程整合：

  • 先確認輸入資料
  • 再進行整理與統計
  • 接著輸出報表
  • 最後補上可閱讀的摘要與驗證步驟

這種「先拆流程、再逐步完成」的方式，會比一開始就追求很大的系統更適合初學者，也更符合真實工作情境。

專案目標

本專案的目標很明確：

  • 讓原始 Excel 能被穩定讀入
  • 自動整理成主管可閱讀的摘要報表
  • 把關鍵結論輸出成文字
  • 保留基礎的錯誤處理與檢查點

這樣的成果不一定很大，但很像職場真實任務，也很適合當作你的第一個整合專案。

專案輸入與輸出

輸入

  • 檔案名稱：weekly_orders.xlsx
  • 主要欄位：地區、金額

輸出

  • 摘要報表：weekly_summary.xlsx
  • 中文摘要：summary.txt
  • 畫面提示：告訴你專案是否完成

建議資料夾結構

project/
├─ weekly_orders.xlsx
├─ weekly_summary.xlsx
├─ summary.txt
└─ report_automation.py

初學者先把資料和程式放在同一層比較簡單，等你熟悉之後再慢慢拆分資料夾。

先拆成小流程

在寫程式前，先把專案拆小：

  1. 確認原始檔案是否存在
  2. 讀取 Excel
  3. 檢查必要欄位是否存在
  4. 依地區統計總金額
  5. 輸出成新的 Excel
  6. 找出金額最高的地區
  7. 產生摘要文字
  8. 把摘要寫入文字檔

這樣做的好處是，如果程式出錯，你比較容易知道卡在哪一段。

程式範例

from pathlib import Path

import pandas as pd

source_file = Path("weekly_orders.xlsx")

if not source_file.exists():
    print("找不到 weekly_orders.xlsx，請先確認檔案是否放在同一個資料夾。")
else:
    df = pd.read_excel(source_file)

    required_columns = {"地區", "金額"}
    if not required_columns.issubset(df.columns):
        print("缺少必要欄位，請確認 Excel 是否包含 地區 和 金額。")
    else:
        summary = df.groupby("地區", as_index=False)["金額"].sum()
        summary = summary.sort_values("金額", ascending=False)

        summary.to_excel("weekly_summary.xlsx", index=False)

        top_region = summary.iloc[0]
        message = (
            f"本週銷售最高地區為 {top_region['地區']}，"
            f"總金額為 {top_region['金額']}。"
        )

        Path("summary.txt").write_text(message, encoding="utf-8")

        print("已輸出 weekly_summary.xlsx")
        print("已輸出 summary.txt")
        print(message)

範例解讀

  • Path("weekly_orders.xlsx") 用來表示原始檔案路徑。
  • exists() 先檢查檔案存不存在，避免一開始就報錯。
  • pd.read_excel() 負責讀取 Excel。
  • required_columns 用來檢查欄位是否齊全。
  • groupby("地區")["金額"].sum() 會把每個地區的金額加總。
  • sort_values() 讓最高金額排在前面，方便直接找到第一名。
  • to_excel() 把統計結果輸出成新檔案。
  • write_text() 把摘要文字寫進文字檔。

這段程式的重點不是炫技，而是讓你看見一個完整的小型工作流程。

專案中的人工流程、Python 流程、AI 協作流程

人工流程

  • 開檔案
  • 檢查欄位
  • 用 Excel 做統計
  • 複製結果
  • 手動寫摘要

Python 流程

  • 自動讀檔
  • 自動分組統計
  • 自動輸出摘要表

AI 協作流程

  • 幫你把專案需求寫成步驟
  • 協助產生第一版程式
  • 解釋每一段在做什麼
  • 幫你補上錯誤處理與延伸功能

最好的合作方式不是「叫 AI 一次全部做完」，而是你先掌握流程，再讓 AI 幫你補速度與說明。

驗收清單

專案完成後，至少要檢查這幾件事：

  1. weekly_orders.xlsx 能成功讀取
  2. 缺檔時會顯示友善訊息
  3. 缺少欄位時會顯示提醒
  4. weekly_summary.xlsx 有正確輸出
  5. summary.txt 有成功建立
  6. 摘要文字與 Excel 統計結果一致

如果這幾項都通過，這個專案就已經有基本可交付性。

常見錯誤與排查

  • 檔案不存在。 先確認 Excel 是否和程式放在同一個資料夾。
  • 欄位名稱不一致。 例如 Excel 寫成「區域」不是「地區」，這時要先改欄位名稱或修改程式。
  • 金額 不是數字。 如果 Excel 裡混入文字，統計前要先清理資料。
  • 摘要檔有輸出，但內容不合理。 先開啟 weekly_summary.xlsx，確認排序和加總是否正確。

練習題

基礎題

  1. 增加一個欄位統計平均金額。
  2. 印出原始資料總筆數。

改寫題

  1. 將摘要檔名改成包含日期，例如 summary_2026-04-24.txt。
  2. 如果資料為空，顯示「本週沒有可統計資料」。

實作題

  1. 將本章程式改成可以同時輸出 Excel 與文字摘要。
  2. 執行後比對輸出的摘要內容，確認與報表數字一致。

小結

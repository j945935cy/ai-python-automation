第 8 章：桌面自動化

這章要解決什麼問題

不是每個系統都有 API，也不是每個工具都能用爬蟲或 Selenium 直接操作。有些舊系統、公司內部工具或 Windows 軟體，只能靠滑鼠、鍵盤和畫面互動。這時你就會遇到桌面自動化。

這一章的重點不是立刻寫出一大段控制滑鼠的程式，而是先學會判斷：

  • 哪些桌面流程適合自動化
  • 哪些步驟需要人工確認
  • 怎麼把螢幕上的操作翻成標準流程

情境案例

假設你每天都要從公司內部報表系統下載資料，流程可能像這樣：

  1. 開啟報表系統
  2. 輸入帳號密碼
  3. 選擇日期區間
  4. 按下載
  5. 將檔案另存到指定資料夾

如果這個系統沒有匯出 API，也無法直接用瀏覽器自動化，那桌面自動化就很有用。

概念說明

桌面自動化是指自動控制滑鼠、鍵盤、視窗或桌面軟體。它常見在舊系統、內部工具或沒有 API 的環境。這類流程最重要的是先寫清楚步驟，因為執行結果很依賴環境。

桌面自動化通常有幾個特性：

  • 很依賴畫面位置
  • 很怕視窗被移動
  • 執行時不適合同時做其他操作
  • 容易受到解析度、縮放比例影響

所以這一章最重要的能力是流程標準化，而不是程式炫技。

人工流程、Python 流程、AI 協作流程

人工流程

  • 打開軟體
  • 點選功能
  • 輸入資料
  • 下載或匯出
  • 檢查結果

Python 流程

  • 將操作步驟標準化
  • 標出每一步的輸入與輸出
  • 找出需要等待或確認的地方
  • 再決定是否值得自動化

AI 協作流程

  • 幫你整理桌面操作步驟
  • 標出高風險與低風險步驟
  • 協助你把需求改寫成自動化規格
  • 先做腳本藍圖，再逐步補成程式

程式範例

steps = [
    "開啟報表系統",
    "登入帳號",
    "輸入日期區間",
    "下載報表",
    "另存檔案"
]

for step in steps:
    print("執行步驟：", step)

範例解讀

  • 這段程式還沒有真的控制滑鼠或鍵盤。
  • 它的作用是先把流程標準化，讓你知道整個任務是由哪些步驟組成。
  • 當流程被列成清單後，你更容易判斷哪一步適合自動化、哪一步該保留人工確認。

這也是桌面自動化最容易被忽略的一點：很多問題不是出在程式，而是流程本身沒有整理好。

輸入與輸出長什麼樣

輸入

  • 一套實際操作步驟
  • 系統畫面上的欄位或按鈕位置
  • 需要輸入的資料，例如日期區間或帳號

輸出

  • 明確的操作腳本
  • 可供 AI 協助產生程式的需求說明
  • 可檢查的下載結果或輸出檔案

常見錯誤與排查

  • 流程步驟太模糊。 例如只寫「下載報表」，但沒有說要先點哪個按鈕、選哪個區間。
  • 執行時移動視窗或滑鼠。 桌面自動化很依賴畫面穩定，執行時最好不要同時做其他操作。
  • 忽略人工確認點。 有些步驟像登入驗證碼或下載完成確認，仍然需要人工介入。
  • 一開始就追求完全自動化。 先讓 60% 到 80% 的流程穩定，比一次做滿全部更實際。

AI 任務

請 AI 根據你的操作步驟，幫你標出哪些步驟適合自動化、哪些地方可能需要人工確認，並整理成一份桌面自動化規格草稿。

AI Prompt 範例

我想做桌面自動化，但我是初學者。
請根據下面步驟，先整理成標準化流程，
再說明哪些步驟適合用 Python 自動化、哪些地方建議保留人工確認：

1. 開啟系統
2. 登入
3. 輸入日期
4. 匯出報表
5. 另存到指定資料夾

請用白話回答，並列出風險提醒。

練習題

基礎題

  1. 寫出一個你每天會做的桌面流程。
  2. 把流程拆成至少 5 個步驟。

改寫題

  1. 標出哪一步最容易失敗，並說明原因。
  2. 標出哪一步應該保留人工確認。

實作題

  1. 把你的流程改寫成 steps = [] 的 Python 清單。
  2. 用 for 迴圈把每一步印出來，模擬執行順序。

挑戰題

設計一個可以交給 AI 的桌面自動化需求說明，讓它能產出第一版腳本藍圖，包括步驟、風險、確認點與預期輸出。

小結

桌面自動化不是先寫程式，而是先讓流程標準化。流程越明確，後續腳本越穩定；風險越早看見，真正執行時就越不容易失控。

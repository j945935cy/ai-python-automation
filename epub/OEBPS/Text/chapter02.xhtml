第 2 章：Prompt 基礎

這章要解決什麼問題

很多人不是不會用 AI，而是「不知道怎麼把需求說清楚」。如果你的描述太模糊，AI 很可能給你一段看起來厲害、實際上不適合初學者的程式。

這一章要教你的，不是神奇咒語，而是一種清楚表達需求的方法。

情境案例

假設你對 AI 說：

幫我整理檔案

這句話太短，AI 會有很多不確定的地方：

  • 要整理哪個資料夾？
  • 是搬移、重新命名，還是分類？
  • 可以使用哪些套件？
  • 你要的是完整程式、步驟說明，還是教學版本？

如果你改成這樣：

你是一位 Python 初學者助教。
請用標準函式庫寫一個簡單程式，
把 downloads 資料夾中的 PDF 搬到 reports 資料夾。
請逐段說明程式用途，並提醒可能的風險。

結果通常就會穩定很多。

概念說明

Prompt 是你交給 AI 的工作說明書。Prompt 越清楚，AI 產出的程式越接近你要的結果。對初學者最有幫助的 Prompt，通常包含四個元素：

  • 角色：請 AI 扮演什麼身份
  • 任務：要完成什麼工作
  • 限制：只能用哪些工具、程式要多簡單
  • 輸出格式：希望回覆有哪些內容

你不一定每次都要寫得很長，但至少要讓 AI 知道你要它做什麼、做到什麼程度。

壞 Prompt 與好 Prompt

壞 Prompt

幫我寫 Python

問題：

  • 不知道要解決什麼工作
  • 不知道要用什麼工具
  • 不知道你是初學者還是進階使用者

好 Prompt

你是一位 Python 初學者助教。
請用 pathlib 寫一個簡單範例，
列出 downloads 資料夾中的所有 PDF 檔名。
請逐行解釋，並避免使用太複雜的寫法。

優點：

  • 有角色
  • 有任務
  • 有限制
  • 有輸出要求

程式範例

prompt = """
你是一位 Python 初學者助教。
請示範如何列出資料夾中的所有檔案。
只能使用標準函式庫，並逐行解釋程式。
"""

print(prompt)

範例解讀

  • 多行字串適合儲存 Prompt，方便修改與重用。
  • Prompt 本身也是教材內容，可以整理成你自己的常用模板。
  • 清楚描述限制，能有效降低 AI 產生過度複雜程式的機率。

人工流程、Python 流程、AI 協作流程

人工流程

  • 自己先想清楚要做什麼
  • 確認輸入資料與預期結果
  • 把需求用白話整理出來

Python 流程

  • 根據明確需求撰寫程式
  • 確認有沒有符合限制
  • 測試輸出結果

AI 協作流程

  • 先請 AI 寫第一版
  • 再請 AI 解釋每一段
  • 如果太難，要求它簡化
  • 如果有錯誤，貼出錯誤訊息請它協助排查

這樣的做法比一次要求 AI 生出「完美答案」更實際。

常見錯誤與排查

  • 需求太模糊。 先補上資料來源、想做的動作、輸出格式。
  • 沒有限制難度。 可以明確寫「適合初學者」「避免進階語法」。
  • 沒有指定工具。 例如你可以寫「只用標準函式庫」或「使用 pandas」。
  • 一次塞太多需求。 先讓 AI 完成第一步，再逐步追加條件，通常更穩。

AI 任務

把你自己寫的一段模糊需求交給 AI，要求它改寫成更適合初學者的高品質 Prompt，並說明改寫理由。

AI Prompt 範例

請把下面這句模糊請求改寫成適合初學者的 Prompt：
「幫我整理檔案」

請補上：
1. 角色
2. 任務
3. 限制
4. 輸出格式

最後請再說明，為什麼原本的需求不夠清楚。

練習題

基礎題

  1. 寫一個 Prompt，要求 AI 幫你列出資料夾中的所有 PDF。
  2. 寫一個 Prompt，要求 AI 用 pandas 讀取 Excel。

改寫題

  1. 把「幫我抓網站資料」改寫成更完整的 Prompt。
  2. 把「幫我修錯誤」改寫成包含錯誤訊息與輸出需求的 Prompt。

實作題

  1. 寫一個完整 Prompt，要求 AI 產生「下載網頁後存成文字檔」的範例程式。
  2. 再追加一句要求 AI 用更白話的方式解釋。

挑戰題

請設計一個你未來會反覆使用的 Prompt 模板，主題不限，但要包含角色、任務、限制與輸出格式四個區塊。

小結

會寫 Prompt，不是因為你很會下命令，而是因為你知道自己真正要什麼。需求越明確，AI 的幫助越穩定。對初學者來說，好的 Prompt 就像清楚的作業說明書。
